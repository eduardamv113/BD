**Detective Agency Database System**

This project was developed for the Databases (Bases de Dados) course.

It follows the complete lifecycle of database creation—from initial requirements gathering to physical implementation in MySQL. The system is designed to manage the operations of a Detective Agency -"Projeto Tricornio"- tracking cases, agents, clients, and evidence.

*Development Phases*

System Definition: Justification for using a Relational Database model.

Requirements Analysis: Defining how data is described, manipulated, and controlled.

Conceptual Model: Designing the Entity-Relationship (ER) Diagram, identifying key entities and their attributes.

Logical & Physical Model: Mapping the ER diagram into relational tables and implementing the schema in SQL.

*Key Features*

Relational Architecture: Organized structure to prevent data redundancy.

Complex Relationships: Handles associations between cases, suspects, and assigned investigators.

SQL Implementation: Full implementation of the physical model using MySQL.
